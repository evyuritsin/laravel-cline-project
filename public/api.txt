# API Документация для работы с AI моделями

## Базовые принципы

**Базовый URL:** `https://api.apifly.ru/api`

**Аутентификация:** Все запросы требуют API ключ, который передается в заголовке `Authorization: Bearer {YOUR_API_KEY}`. (template_token)

**Формат запросов и ответов:** JSON.

**Коды ответов:**
- `200` - Успешный запрос
- `400` - Ошибка валидации
- `402` - Недостаточно средств на балансе (Payment Required)
- `500` - Внутренняя ошибка сервера

**Универсальный метод проверки статуса задачи** (описан в конце документации).

## Модели генерации изображений

### 1. GPT Image 1.5

#### 1.1. Text to Image
Создание изображения по текстовому описанию.

**Эндпоинт:** `POST /models/gpt-image/1.5-text-to-image`

**Параметры запроса:**

| Параметр | Тип | Обязательный | Описание | Допустимые значения |
|----------|-----|--------------|----------|---------------------|
| `prompt` | string | Да | Текстовое описание | До 3000 символов |
| `aspect_ratio` | string | Да | Соотношение сторон | `1:1`, `2:3`, `3:2` |
| `quality` | string | Да | Качество изображения | `medium`, `high` |
| `callback_url` | string | Нет | URL для вебхука | Действительный URL |

**Система ценообразования:**

| Качество | Стоимость ($) |
|----------|---------------|
| `medium` | 0.03 |
| `high` | 0.17 |

**Пример запроса:**
```json
{
  "prompt": "A beautiful sunset over mountains, digital art",
  "aspect_ratio": "16:9",
  "quality": "high",
  "callback_url": "https://your-server.com/webhook"
}
```

#### 1.2. Image to Image
Создание изображения на основе существующего изображения и текстового описания.

**Эндпоинт:** `POST /models/gpt-image/1.5-image-to-image`

**Параметры запроса:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `input_urls` | array | Да | Массив URL изображений (минимум 1) |
| `prompt` | string | Да | Текстовое описание (до 3000 символов) |
| `aspect_ratio` | string | Да | Соотношение сторон (`1:1`, `2:3`, `3:2`) |
| `quality` | string | Да | Качество (`medium`, `high`) |
| `callback_url` | string | Нет | URL для вебхука |

**Система ценообразования:** Аналогично Text to Image.

## Универсальный метод получения статуса задачи

**Эндпоинт:** `GET /models/status/{taskId}`

**Параметры URL:**
- `taskId` (string, обязательный) - ID задачи, полученный при создании.

**Описание:** Позволяет проверить статус выполнения любой задачи (изображение, видео, аудио).

**Пример ответа (успешный):**
```json
{
  "success": true,
  "data": {
    "taskId": "12345",
    "status": "completed",
    "resultJson": {
      "resultUrls": ["https://example.com/result.png"]
    },
    "param": {
      "prompt": "A beautiful sunset"
    }
    // ... другие поля
  }
}
```

**Пример ответа (в процессе):**
```json
{
  "success": true,
  "data": {
    "taskId": "12345",
    "status": "processing",
    "progress": 45
  }
}
```

**Пример ответа (ошибка):**
```json
{
  "success": false,
  "message": "Task not found"
}
```

## Вебхуки (Callbacks)

Если при создании задачи был передан параметр `callback_url`, по завершении обработки отправит POST-запрос на указанный URL с результатами.

**Пример тела вебхука:**
```json
{
  "taskId": "12345",
  "status": "completed",
  "resultUrls": ["https://example.com/result.png"],
  "param": {
    "prompt": "A beautiful sunset"
  }
}
```

Ваш сервер должен ответить статусом `200 OK`, чтобы подтвердить получение.
